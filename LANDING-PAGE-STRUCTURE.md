# Repository Landing Page Structure

Use this order to maximize first-visit clarity and download conversion.

## 1. First screen

Show immediately:
- project name
- one-line value proposition
- supported operating system
- release download link

Recommended structure:
- `X Recorder`
- `Open-source Windows screen recorder for fullscreen, window, microphone, system audio, and audio-only capture.`
- `Windows 10/11 64-bit`
- `Download latest release`

## 2. Main proof section

Directly after the intro, show:
- main screenshot
- short feature bullets

Users should understand within a few seconds:
- what the app looks like
- what it records
- that it is already usable

## 3. Download section

Show both artifacts clearly:
- installer
- portable zip

Each should have one short explanation:
- installer for most users
- portable zip for users who do not want installation

## 4. Feature section

Keep this simple and scannable:
- fullscreen recording
- window recording
- audio-only recording
- microphone capture
- system audio capture
- Chinese / English UI

## 5. Screenshot section

Recommended order:
1. Main screen
2. Fullscreen example
3. Window recording example
4. Recording control window
5. Countdown window
6. Support author window

## 6. Trust section

State clearly:
- open-source under GPL-3.0
- public source code on GitHub
- current Windows signing status

Recommended note:
- `The app is open source under GPL-3.0. Current Windows releases are unsigned, so SmartScreen may show a warning on first launch.`

## 7. Support section

Only place this after users already understand the product.

Suggested wording:
- `If X Recorder is useful to you, you can support the author voluntarily inside the app.`

## 8. Technical notes

Put these lower on the page:
- Windows 10/11 only
- possible driver / policy limitations
- portable folder must stay intact

## 9. Privacy and license

Link both near the bottom:
- privacy
- license

## 10. Keep above-the-fold focused

Do not put these too early:
- long development notes
- donation screenshots
- detailed technical caveats
- too many badges
