# Post-Release Checklist

## Immediately after release

- [ ] Confirm the GitHub repository page opens normally
- [ ] Confirm the release page opens normally
- [ ] Confirm `X-Recorder-Setup.exe` downloads correctly
- [ ] Confirm `X-Recorder-Portable.zip` downloads correctly
- [ ] Confirm the README renders correctly on GitHub
- [ ] Confirm the GPL-3.0 license is detected by GitHub

## Presentation improvements

- [ ] Upload real screenshots to the repository and download pages
- [ ] Add the release cover image as social preview / listing cover
- [ ] Keep the repository description and topics accurate
- [ ] Keep release notes clear and updated

## Trust improvements

- [ ] Plan Windows code signing
- [ ] Keep release artifacts aligned with the public source code
- [ ] Keep privacy and license documents visible
- [ ] Revoke temporary publishing tokens when no longer needed

## Growth and discoverability

- [ ] Publish an itch.io page
- [ ] Share the project on relevant forums or communities
- [ ] Post an English short announcement
- [ ] Post a Chinese short announcement
- [ ] Track which platform brings the most downloads

## Maintenance

- [ ] Fix reported issues
- [ ] Publish follow-up releases
- [ ] Keep the project active so it looks maintained
- [ ] Improve screenshots and project presentation over time

## Optional future steps

- [ ] Apply for open-source signing support
- [ ] Evaluate Microsoft Store distribution
- [ ] Improve installer trust with code signing
