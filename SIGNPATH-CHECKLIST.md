# SignPath Foundation Checklist

Use this checklist before applying for free code signing support for open-source software.

## Project status

- [x] Public GitHub repository
- [x] Public release available for download
- [x] Standard OSI-approved open-source license
- [x] License detected by GitHub
- [x] Project has a clear README
- [x] Project has an identifiable maintainer
- [x] Software is intended for public use

## Recommended improvements before applying

- [ ] Add real product screenshots to the repository and download page
- [ ] Keep publishing updates so the project shows maintenance activity
- [ ] Improve the repository social preview / cover image
- [ ] Make the project description and release notes more complete over time
- [ ] Keep download links current and public

## Things to keep clean

- [ ] Do not mix public open-source release claims with a closed-source license
- [ ] Do not remove the GPL-3.0 license from the repository
- [ ] Keep the main public build aligned with the public source code
- [ ] Keep project metadata accurate on GitHub
